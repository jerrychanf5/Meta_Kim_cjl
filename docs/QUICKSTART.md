# Meta_Kim Quickstart

Meta_Kim quickstart guide.

## 1. Project Overview

**Meta_Kim** is an AI governance system that gives Claude Code, Codex, OpenClaw, Cursor, and similar coding assistants one shared governance layer.

Core loop:

> clarify what needs to be done -> decide who should do it -> review after execution -> retain learning -> feed the next run.

Short version: Meta_Kim is governance for AI work so AI execution does not drift out of control.

## 2. Install In 3 Steps

### Option A: Interactive Install

```bash
git clone https://github.com/KimYx0207/Meta_Kim.git
cd Meta_Kim
npm install
node setup.mjs
```

Or use npm:

```bash
npm install -g meta-kim
meta-kim setup
```

### Option B: Sync Runtimes From A Cloned Repository

```bash
npm install
npm run meta:sync          # sync canonical -> .claude/ .codex/ .cursor/ openclaw/
npm run meta:validate      # validate project integrity
```

### Option C: Try With npx

```bash
npx meta-kim setup
```

## 3. Basic Usage

### Example 1: Sync Runtime Files

After editing agents or skills under `canonical/`, sync the runtime mirrors:

```bash
npm run meta:sync
```

### Example 2: Run Governance Self-Check

Check contract shape, hook commands, and mirror sync:

```bash
npm run meta:check
```

### Example 3: Validate A Run Artifact

Test whether a local governance record is valid:

```bash
npm run meta:validate:run -- tests/fixtures/run-artifacts/valid-run.json
```

For more commands, inspect `package.json` scripts or run:

```bash
npm run
```

## 4. Memory Activation Layers

| Layer | Source | Trigger | Status |
| --- | --- | --- | --- |
| **Layer 1** | `canonical/` sources for agents, skills, and runtime assets | after canonical files change | requires `meta:sync` |
| **Layer 2** | runtime directories such as `.claude/` and `.codex/` | after `meta:sync` runs | generated automatically |
| **Layer 3** | run-index and local profile state | after `meta:index:runs` indexes records | queryable |

Layer 2 content is generated by `meta:sync` and should not be hand-edited because it will be overwritten. Layer 3 requires an explicit `meta:index:runs` call.

## 5. Common Issues

### Q1: `meta:sync` reports "hooks out of sync"

Run:

```bash
npm run meta:check
```

Then confirm the contract and hook commands match, and that `.claude/settings.json` hooks align with `scripts/validate-project.mjs`.

### Q2: Node version is unsupported

```bash
node --version   # requires >= 22.13.0
```

Upgrade Node.js and retry.

### Q3: `meta:validate:run` reports `ok: false`

Check that the artifact path is correct and use a repository-relative path:

```bash
npm run meta:validate:run -- tests/fixtures/run-artifacts/valid-run.json
```

### Q4: `meta:doctor` script is missing

Confirm you are in the repository root and that `node setup.mjs` completed successfully.

### Q5: run-index query returns empty results

Index runs first:

```bash
npm run meta:index:runs -- runs/
npm run meta:query:runs -- --owner meta-warden
```
